    Pytorch Pipeline#

    记录一下写 Pytorch 的代码时的标准流程和模版

    Argparse#

    Argparse的 入门介绍，Argparse 是 Python 标准库中推荐的命令行解析模块。是在运行代码时，给代码中的一些函数增加可修改性

    可能会需要设置的参数：

    import argparse
parser = argparse.ArgumentParser()
parser.add_argument('--batch_size', type=int, default=64, help='batch size')
parser.add_argument('--drop_out_rate', type=float, default=0.5, help='drop out probability')
parser.add_argument('--gpu', type=int, default=0, help='gpu device id')
parser.add_argument('--learning_rate', type=float, default=0.001, help='learning rate')
parser.add_argument('--seed', type=int, default=2, help='random seed')
parser.add_argument('--max_epochs', type=int, default=50, help='num of training epochs')
parser.add_argument('--model_path', type=str, default='saved_models', help='path to save the model')
parser.add_argument('--momentum', type=float, default=0.9, help='momentum')
args = parser.parse_args()
print(args.batch_size) # 64

    在运行程序时修改 parser 内的参数，比如修改 batch_size 为128

    python XXX.py --batch_size 128

    如果用 jupyter notebook 去设置 arparse 需要额外添加的语句，否则会报错

    import sys

    Attentions:#

    1.不要使用太大的 nn.Linear(m, n) 层，因为这本质是矩阵乘法，空间复杂度是 \mathcal{O}(mn) 的，linear 层太大会非常吃显存

    2.不要在太长序列中使用 RNN，因为 RNN 使用的是 BPTT 算法，其需要的内存和输入序列的长度呈线性相关

    3.在训练的之前调用 model.train()，在测试的之前调用 model.eval()

    4.不需要梯度的代码块用 with torch.no_grad() 包含起来

    5.model.eval() 和 torch.no_grad() 的区别是，model.eval() 是将网络切换成测试状态例如 batch normalization 和 dropout 在训练和测试的算法是不一样的；torch.no_grad() 则是将所有 tensor 的自动求导机制关闭，以减少存储使用和加速计算，得到的结果无法进行 loss.backward()

    6.loss.backward() 前用 optimizer.zero_grad() 将积累的梯度清零

    7.model.zero_grad() 会将整个网络的参数的梯度清零；optimizer.zero_grad() 只是将传入优化器的参数清零

    8.torch.utils.data.DataLoader 中尽量设置 pin_memory=True，对特别小的数据集如 MNIST 设置 pin_memory=False 反而更快一些。num_workers 的设置需要在实验中找到最快的取值。num_workers 是处理 dataloader 的线程数目，并不一定越多越好
